Halifax Scooter Council (In Development)

The Halifax Scooter Council is a group of scooterists who meet to organize events, group rides, and a yearly rally. In previous years, the organization was smaller and events were planned by word-of-mouth. As the group grew larger, a centralized place for meets was needed, and the club launched a website and began using Facebook to communicate.

Unfortunately, some of the members do not use Facebook, the club "leaders" are not extremely adept Facebook users, and would prefer a stronger web presence. I joined the club, and decided about one month ago to undertake redoing the site from a series of static HTML pages that were rarely updated to a dynamic Rails site that allows users to upload their photo and pictures of their scooters, post events/group rides, and send to friends to promote the Rally. The site is undergoing some minor tweaks and design touches while the domain transfers over to me; I plan to launch the new site on July 18.

Prey Fetcher

I'm a pretty avid Twitter user — something like a bad habit. I've found the service useful, but I lamented that there was no way for me to get notified of mentions when I was away from my computer without running a service on my computer. I didn't like that solution; sometimes my machine is off, and figured others with laptops were in the same boat.

To solve the problem, and to get my feet wet with Ruby on Rails, I created Prey Fetcher, a free service that pushes mentions to your iPhone (or iPod Touch/iPad) instantly. It also watches Twitter lists and direct messages and polls the REST API to deliver those notifications in a timely manner. The user interface is simple; just sign-in with your Twitter account using their OAuth system and an account is created in the Prey Fetcher system.

In May of this year, I open-sourced Prey Fetcher, so that others could benefit from its code and hopefully add features; many of its users seem to be of a technical nature, based on discussions I've had with them.

Technologies Used/Implemented:

  • Apache 2, PostgreSQL (later MySQL), lighttpd, Ruby on Rails 2.3
  • OAuth authentication
  • Streaming API daemon built with Ruby EventMachine
  • 1,000 of users with tonnes of room to scale on a $5/month VPS

FristPost (In Development)

I've been interested in starting to publish articles online, but was not content with the popular blogging software that existed; WordPress and Drupal were too clunky to me, and I especially didn't like their code (this mattered, as I'd be writing about software development).

In the time-honoured tradition, I set out to write the best piece of blogging software ever — not really, but I started working on a CodeIgniter app called FristPost that had a more flexible license (BSD) and would be easy for developers to build on without having every feature ever. It was my first CodeIgniter app, and is currently under development.

FristPost is an open-source app.

Technologies Used/Implemented:

  • Database-agnostic, thanks to CodeIgniter's ActiveRecord
  • Simple key/value cache store system that has multiple drivers (Filesystem, Database, Memcache, and "null" — for development)
  • MVC with thin views thanks to Smarty templates
  • Emphasis on performance; over twice as fast as WordPress, even in development mode

Nova Scotia Government and Nova Scotia Come to life

I've worked heavily with the provincial Government, including the Come to life initiative, to improve the accessibility, best practices, markup, and progressiveness of their sites and web applications. Much of the output of this work involves dealing with vendors and providing them with a web standards guide that I was a part in creating, and building an ASP framework that web developers of varying skill across government can use to easily create websites sharing a common look-and-feel.

I have moved numerous services and sites from propriety software to free and open-source software to save money, improve quality, and lower barrier-to-entry for external vendors. This includes moving the Come to life site to the Drupal CMS and developing new internal web apps to use databases and PHP/Rails instead of email/ASP. The main government site still runs on Windows, though I have suggested a reverse-HTTP proxy server to free developers of the Windows platform. This idea has been well-received.

My most recent project in government was to implement best practices in terms of software development, including the use of git for version control, PHP and Ruby frameworks for web development, status/issue tracking with Redmine, and app deployment with vlad.
